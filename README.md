## ✨ Why NeuralMind?

When working with AI coding assistants like Claude, GPT-4, or Cursor, you face a fundamental challenge: **context window limits**. Loading an entire codebase consumes 50,000+ tokens, leaving little room for meaningful conversation.

NeuralMind solves this by creating **intelligent, query-aware context** from your codebase knowledge graphs. Instead of dumping everything into the prompt, it loads only what's relevant—achieving massive token savings while maintaining understanding.

### Key Features

| Feature | Description |
|---------|-------------|
| 🚀 **Massive Token Reduction** | 40-70x fewer tokens than loading full codebases |
| 📊 **4-Layer Progressive Disclosure** | Only load what's relevant to your query |
| 🔍 **Semantic Search** | Find code entities by meaning, not just keywords |
| 🏘️ **Community Awareness** | Understands logical code clusters and relationships |
| 🔌 **MCP Integration** | Works with Claude Desktop, Cursor, and other MCP tools |
| ⚡ **Incremental Updates** | Only re-embed changed nodes for fast rebuilds |

## 🚀 Quick Start

### 1. Generate Knowledge Graph

First, create a knowledge graph for your project using [graphify](https://github.com/safishamsi/graphify):

```bash
pip install graphifyy
graphify update /path/to/your/project
```

This creates `graphify-out/graph.json` with your codebase structure.

### 2. Build Neural Index

```bash
neuralmind build /path/to/your/project
```

### 3. Query Your Codebase

```bash
# Get wake-up context for starting a conversation
neuralmind wakeup /path/to/your/project

# Query with natural language
neuralmind query /path/to/your/project "How does authentication work?"
```

### Python API

```python
from neuralmind import NeuralMind

# Initialize for your project
mind = NeuralMind('/path/to/project')
mind.build()  # Generate embeddings from graph.json

# Wake-up context (~600 tokens) for starting a conversation
wakeup = mind.wakeup()
print(wakeup.context)
print(f"Tokens: {wakeup.budget.total}")

# Query context (~1000-1500 tokens) for specific questions
result = mind.query("How does authentication work?")
print(result.context)
print(f"Token reduction: {result.reduction_ratio:.1f}x")
```

## 🏗️ Architecture

NeuralMind uses a **4-layer progressive disclosure** architecture to minimize token usage while maximizing relevance:

```
┌─────────────────────────────────────────────────────────────────┐
│                        NEURALMIND ARCHITECTURE                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │  L0: IDENTITY LAYER                        (~100 tokens) │    │
│  │  ├── Project name and description                        │    │
│  │  ├── Key facts and metadata                              │    │
│  │  └── Always loaded                                       │    │
│  └─────────────────────────────────────────────────────────┘    │
│                              ▼                                   │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │  L1: SUMMARY LAYER                         (~500 tokens) │    │
│  │  ├── High-level architecture overview                    │    │
│  │  ├── Main components and code clusters                   │    │
│  │  └── Always loaded                                       │    │
│  └─────────────────────────────────────────────────────────┘    │
│                              ▼                                   │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │  L2: ON-DEMAND LAYER                   (~200-500 tokens) │    │
│  │  ├── Specific modules relevant to query                  │    │
│  │  ├── Community/cluster details                           │    │
│  │  └── Loaded per query                                    │    │
│  └─────────────────────────────────────────────────────────┘    │
│                              ▼                                   │
│  ┌─────────────────────────────────────────────────────────┐    │
│  │  L3: SEARCH LAYER                      (~200-500 tokens) │    │
│  │  ├── Semantic search results                             │    │
│  │  ├── Relevant code snippets                              │    │
│  │  └── Loaded per query                                    │    │
│  └─────────────────────────────────────────────────────────┘    │
│                                                                  │
├─────────────────────────────────────────────────────────────────┤
│  TOKEN BUDGET                                                    │
│  ├── Wake-up: L0 + L1 = ~600 tokens                             │
│  ├── Query: L0 + L1 + L2 + L3 = ~1000-1500 tokens               │
│  ├── Full codebase: 50,000+ tokens                              │
│  └── Reduction: 40-70x typical                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 🔌 MCP Server Integration

NeuralMind includes a Model Context Protocol (MCP) server for integration with Claude Desktop, Cursor, and other MCP-compatible tools.

### Starting the Server

```bash
# Using the CLI
neuralmind-mcp

# Or as a Python module
python -m neuralmind.mcp_server
```

### Claude Desktop Configuration

Add to your `claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "neuralmind": {
      "command": "neuralmind-mcp",
      "args": []
    }
  }
}
```

### Available MCP Tools

| Tool | Description |
|------|-------------|
| `neuralmind_wakeup` | Get wake-up context for a project |
| `neuralmind_query` | Query project with natural language |
| `neuralmind_search` | Semantic search across codebase |
| `neuralmind_build` | Build/rebuild neural index |
| `neuralmind_stats` | Get project statistics |
| `neuralmind_benchmark` | Run performance benchmark |

## 📚 API Reference

### NeuralMind Class

```python
class NeuralMind:
    """Main interface for neural codebase understanding."""
    
    def __init__(self, project_path: str, db_path: str = None):
        """
        Initialize NeuralMind for a project.
        
        Args:
            project_path: Path to project root (must contain graphify-out/graph.json)
            db_path: Optional custom path for ChromaDB storage
        """
    
    def build(self, force: bool = False) -> Dict:
        """
        Build neural index from graph.json.
        
        Args:
            force: If True, re-embed all nodes even if unchanged
            
        Returns:
            Dict with build statistics (nodes_processed, time_elapsed, etc.)
        """
    
    def wakeup(self) -> ContextResult:
        """
        Get minimal wake-up context for starting a conversation.
        Loads L0 (Identity) + L1 (Summary) layers.
        
        Returns:
            ContextResult with ~600 tokens of project context
        """
    
    def query(self, question: str) -> ContextResult:
        """
        Get optimized context for a specific question.
        Loads all 4 layers with query-relevant content.
        
        Args:
            question: Natural language question about the codebase
            
        Returns:
            ContextResult with ~1000-1500 tokens of relevant context
        """
    
    def search(self, query: str, n: int = 10, **filters) -> List[Dict]:
        """
        Direct semantic search across codebase entities.
        
        Args:
            query: Search query (semantic, not keyword-based)
            n: Maximum number of results
            **filters: Optional filters (node_type, community, etc.)
            
        Returns:
            List of matching entities with scores and metadata
        """
    
    def benchmark(self, sample_queries: List[str] = None) -> Dict:
        """
        Run performance benchmark with sample queries.
        
        Args:
            sample_queries: Optional custom queries; uses defaults if None
            
        Returns:
            Dict with benchmark results (avg_tokens, avg_reduction, etc.)
        """
    
    def export_context(self, query: str = None, output_path: str = None) -> str:
        """
        Export context to file for debugging or manual use.
        
        Args:
            query: Optional query; exports wake-up context if None
            output_path: Optional file path; returns string if None
            
        Returns:
            Context string or path to exported file
        """
    
    def get_stats(self) -> Dict:
        """
        Get statistics about the neural index.
        
        Returns:
            Dict with node_count, community_count, embedding_dimensions, etc.
        """
```

### ContextResult

```python
@dataclass
class ContextResult:
    """Result of context generation."""
    
    context: str
    """The optimized context text ready for AI consumption."""
    
    budget: TokenBudget
    """Token usage breakdown by layer."""
    
    layers_used: List[str]
    """Which layers were loaded (e.g., ['L0', 'L1', 'L2', 'L3'])."""
    
    communities_loaded: List[int]
    """IDs of code clusters/communities that were accessed."""
    
    search_hits: int
    """Number of semantic search results included."""
    
    reduction_ratio: float
    """Token reduction compared to full codebase (e.g., 50.0 = 50x reduction)."""
```

### TokenBudget

```python
@dataclass
class TokenBudget:
    """Token usage tracking across layers."""
    
    l0: int  # Identity layer tokens
    l1: int  # Summary layer tokens
    l2: int  # On-demand layer tokens
    l3: int  # Search layer tokens
    total: int  # Total tokens used
```

## 🔧 Configuration

NeuralMind automatically detects project configuration from:

1. `mempalace.yaml` - Full configuration file
2. `CLAUDE.md` - Project description for AI assistants
3. `README.md` - Fallback for project name/description

### Example mempalace.yaml

```yaml
project:
  name: "My Awesome Project"
  description: "A revolutionary system for doing amazing things"
  wing: "awesome-project"

neuralmind:
  # Token budgets per layer
  token_budget:
    l0: 100
    l1: 500
    l2: 500
    l3: 500
  
  # Search parameters
  search:
    default_limit: 10
    similarity_threshold: 0.7
```
